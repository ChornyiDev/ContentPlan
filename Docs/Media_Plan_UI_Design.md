# Media Plan UI: Design Proposals

Оскільки ми відмовилися від формату суцільної таблиці на користь ієрархічної структури (Campaign -> Budget Block -> Segment), ось кілька підходів до дизайну UI для сторінки **Media Plan**, яка буде доступна всередині Content Plan.

---

## Варіант А: "Kanban / Block" стиль (Рекомендовано)

Цей стиль ідеально підходить для SaaS-інтерфейсів і добре лягає на структуру FlutterFlow (List -> Container -> List -> Container).

### Як це виглядає:
1.  **Верхня панель (Campaign Info):**
    *   Dropdown для вибору Кампанії (оскільки Media Plan прив'язаний до кампанії).
    *   **Total Budget Widget:** Великі цифри загального бюджету, який автоматично рахується з усіх блоків нижче.
    *   Кнопка `+ Add Budget Block`.
2.  **Budget Blocks (Картки, які йдуть зверху вниз):**
    *   Кожен Бюджет — це велика візуальна картка (Container).
    *   **Шапка картки:** Назва платформи (Meta, TikTok), сума бюджету на цей блок, і кнопка `Edit / Delete`.
    *   **Тіло картки (Custom Fields):** Легкі сірі плашки (Chip або маленькі Container) у рядок: `KPI: Sales`, `Region: EU`.
    *   **Тіло картки (Segments):** Всередині картки бюджету йде список (ListView/Column) дочірніх сегментів.
3.  **Segment Rows (Всередині Budget Block):**
    *   Кожен сегмент — це рядок (Row) з білим фоном і легкою тінню або бордером.
    *   **Ліва частина:** Назва сегменту (напр., "Audience 1 + Lookalike"), кастомні поля сегменту (напр., Placement: Stories).
    *   **Права частина (Графік):** 52 маленькі квадратики (або 12 місяців, якщо тижні занадто дрібні), які клікабельні. Активні підсвічуються брендовим кольором (наприклад, #0369A1 або Emerald для успішного статусу).

### Плюси:
*   Дуже сучасний вигляд.
*   Легко адаптується під мобільні екрани (рядки перетворюються на колонки).
*   Контент структурований: користувач чітко бачить, які сегменти належать до якого бюджету.

---

## Варіант B: "Data Table with Expandable Rows" (Гібридний стиль)

Цей варіант намагається зберегти звичний для таблиць вигляд, але робить його "розумним".

### Як це виглядає:
1.  **Основна таблиця — це Бюджети:**
    *   Колонка 1: Платформа
    *   Колонка 2: Бюджет
    *   Колонка 3: Custom Fields (через кому або у вигляді Chips)
    *   Колонка 4: Дія (Розгорнути/Згорнути)
2.  **Розгорнутий стан (Segments):**
    *   При кліку на Бюджет рядок "розкривається" вниз.
    *   З'являється вкладена таблиця Сегментів для цього бюджету.
    *   Вкладена таблиця має колонки: Segment Name, Placements (кастомні поля), і широку колонку **Timeline** (де намальовані клікабельні тижні).

### Плюси:
*   Більш звично для тих, хто роками працював в Excel.
*   Займає менше місця по вертикалі в згорнутому стані.

### Мінуси:
*   У FlutterFlow трохи складніше реалізувати якісні "розумні таблиці" з вкладеністю (expandable data tables), ніж просто списки з контейнерами.
*   Графік на 52 тижні все одно буде складно вмістити поруч із даними сегмента на маленьких екранах.

---

## Особливості реалізації "Графіка по тижнях" (Timeline)

Яку б структуру (A чи B) ми не обрали, нам потрібно красиво відобразити ці **52 зелені клітинки з Excel**. Замість того, щоб показувати 52 квадратики в ряд, пропоную такий варіант для UI:

**Компактний Timeline:**
Показувати смужку з 12 місяців: `[Jan] [Feb] [Mar] [Apr] ... [Dec]`
Коли користувач наводить курсор (або клікає) на місяць, він "розкривається" у 4-5 квадратиків тижнів для вибору.

АБО

**Horizontal Scroll Timeline:**
Рядок постійної висоти з горизонтальним скролом (ListView(scrollDirection: Axis.horizontal)), всередині якого лежать згруповані по місяцях тижні. Це дозволить зберегти точність Excel, не ламаючи верстку екрану на менших моніторах.

## Що я рекомендую?

Я настійно рекомендую **Варіант А (Kanban / Block)** із **Horizontal Scroll Timeline** для сегментів.
1. Це найпростіше і найчистіше зібрати у FlutterFlow.
2. Це дає "Wow"-ефект для B2B клієнтів (виглядає як сучасний Notion / Asana, а не як старий Excel).
3. Це дозволяє легко додавати "Custom Fields" просто як плашки (Chips) без зламу колонок.

Як вам Варіант А? Якщо він підходить, я можу створити детальний HTML/CSS прототип цієї сторінки (як ми робили раніше для інших екранів), щоб ви побачили це вживу.
